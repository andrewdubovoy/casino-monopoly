# 🎲 Monopoly-Style Trading Game

Браузерная мультиплеерная игра в стиле монополии с торговыми механиками, боссом и системой задач. Тёмная золотая тема, полностью адаптивный интерфейс.

---

## 🚀 Запуск

```bash
node server.js
```

Приложение доступно на `http://localhost:3000` (или `PORT` из env).  
Статика раздаётся из папки `public/`.

---

## 🗂 Структура проекта

```
├── server.js        — Node.js HTTP-сервер (без фреймворков)
├── public/
│   ├── index.html   — основной HTML
│   ├── styles.css   — все стили (~4000 строк)
│   └── Scale.js     — адаптивное масштабирование
```

## 🎮 Игровой процесс

### Экраны

| # | Экран | Описание |
|---|-------|----------|
| 1 | **VS-экран** | Выбор и отображение двух игроков с аватарами |
| 2 | **Стартовый капитал** | Ввод начального депозита (быстрые кнопки + ручной ввод в рублях) |
| 3 | **Игровое поле** | Основной геймплей — доска, кубики, панели игроков |
| — | **Win / Lose** | Экраны победы и поражения с анимацией конфетти |

---

## 🏟 Игровое поле

- Доска **11×11** клеток (сетка `grid-template-columns: 80px repeat(9, 1fr) 80px`)
- **36 клеток** по периметру + угловые клетки
- Клетки разделены по **цветовым группам** (красный, оранжевый, зелёный, синий, фиолетовый и др.) с цветной полосой-маркером
- В центре доски — панель управления с кубиками и кнопкой броска

### Угловые клетки

- **Старт / GO** — начальная позиция, круговой обход даёт бонус
- **Парковка** — нейтральная зона
- **Тюрьма** и **Иди в тюрьму** — классические механики

---

## 🎲 Кубики и ход

- Бросок **2 кубиков** с анимацией тряски (`die-shake`)
- После приземления — анимация свечения (`die-land`)
- Если выпадает **дубль** — активируется оверлей `DOUBLE!` с особым эффектом и бонусом к мультипликатору
- Токен игрока плавно перемещается по доске (`transition` на `left/top`)

---

## 📋 Задания на клетках

При попадании на клетку открывается **Cell Modal** с:
- Иконкой и названием клетки
- Описанием задания
- Ценой выполнения (в рублях)
- Кнопками **Принять** / **Отклонить**

Отклонение задания снимает **жизнь** с игрока.

---

## ❤️ Жизни

- У каждого игрока есть **жизни** (отображаются сердечками на панели)
- При отклонении задания сердце анимированно гаснет (`heartLose`)
- Потеря всех жизней → экран поражения

---

## 💰 Деньги и мультипликатор

- Баланс каждого игрока отображается на боковой панели в рублях (или `∞` при безлимитном режиме)
- **Multiplier Widget** — виджет в правом нижнем углу, показывает текущий множитель выплат
- Множитель накапливается за счёт:
  - Прохождения круга (**Lap бонус**)
  - Выпадения дублей (**Double бонус**)
  - Штрафов (`penalty`)

---

## 👹 Босс

- Один из игроков может быть помечен как **BOSS** (красный бейдж на аватаре)
- В нижнем правом углу — **Boss Bar**: полоска HP босса
- При выполнении задания против босса HP уменьшается, полоска обновляется с анимацией
- При попадании: анимация удара (`bossHit`) + всплывающий **damage popup** с числом урона
- Когда HP босса обнуляется — триггерится победный экран

---

## 🔄 Круги (Laps)

- При прохождении полного круга появляется **Lap Notification** (тост снизу экрана)
- Зелёный тост — бонус получен, красный — штраф
- Круги увеличивают мультипликатор

---

## 🏳️ Сдаться

- Кнопка **Surrender** (красная, внизу слева) позволяет игроку сдаться досрочно
- Доступна только во время активной игры

---

## 🏆 Конец сессии

После завершения игры открывается **Session Overlay** с:
- Итоговой статистикой (6 метрик в сетке): ходы, доход, урон по боссу, победы и др.
- **Подиумом** топ-3 задач по ROI с анимированными барами
- Кнопками **Сыграть снова** / **Закрыть**

---

## 💸 Додеп (докупить депозит)

На экране поражения доступна кнопка **Додеп** — позволяет игроку добавить деньги и продолжить игру вместо полного рестарта. Подтверждение через отдельный диалог с зелёным тостом после успеха.

---

## 📐 Адаптивное масштабирование (`Scale.js`)

- Базовый размер интерфейса: **1440 × 900 px**
- `Scale.js` вычисляет `scale = min(vw/1440, vh/900)` и применяет `transform: scale()` на `<body>`
- Работает при ресайзе окна, изменении зума (`Ctrl+/-`) и через `ResizeObserver`
- Нет горизонтального/вертикального скролла — интерфейс всегда вписан в экран

## 🎨 Дизайн-система

- **Палитра**: тёмный фон `#141414`, акцент золото `#f5a623`, красный `#e63946`, зелёный `#2dc653`
- **Шрифт**: Montserrat (900 для заголовков, 700–800 для UI)
- **Частицы**: 12 анимированных золотых частиц на фоне (`float-up`)
- **Переходы экранов**: `slideInLeft / Right`, `fadeInUp`, `cubic-bezier(.22, 1, .36, 1)`
- **Telegram-баннер**: QR-код в правом нижнем углу со ссылкой на канал
